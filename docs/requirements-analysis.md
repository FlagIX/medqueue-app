# MedQueue 医疗预约挂号平台 — 需求分析表

| 模块 | 功能点 | 需求描述 | 优先级 | 备注 |
|------|--------|----------|--------|------|
| **用户** | 手机验证码登录 | 输入手机号获取验证码，验证码校验登录 | P0 | 复用原逻辑 |
| 用户 | 自动注册 | 首次登录自动创建账号 | P0 | 复用 |
| 用户 | Token认证 | Redis存储登录态，token返回前端 | P0 | 复用 |
| 用户 | Token自动刷新 | 拦截器每次请求自动刷新token有效期 | P0 | 复用 |
| 用户 | 登出 | 清除Redis中的token | P2 | 原项目未实现 |
| **就诊人** | 添加就诊人 | 填写姓名、身份证、手机号、关系 | P0 | 新增 |
| 就诊人 | 就诊人列表 | 查看已添加的就诊人 | P0 | 新增 |
| 就诊人 | 删除就诊人 | 删除就诊人信息 | P1 | 新增 |
| **科室** | 科室列表 | 按排序返回科室列表（Redis List缓存） | P0 | 复用原ShopType |
| **医院** | 医院列表分页 | 按科室筛选、分页查询医院 | P0 | 改造原Shop |
| 医院 | 医院详情 | Redis三级缓存（穿透/击穿处理） | P0 | 复用CacheClient |
| 医院 | 附近医院 | Redis GEO按距离排序搜索 | P1 | 复用现有经纬度字段 |
| 医院 | 医院评分更新 | 评价后自动更新评分和评价数 | P1 | 新增逻辑 |
| **医生** | 医生列表 | 按医院/科室筛选，分页查询 | P0 | 新增实体 |
| 医生 | 医生详情 | 医生个人信息+累计挂号数 | P0 | 新增 |
| 医生 | 排班视图 | 按医生+日期查看号源情况 | P0 | 改造原秒杀券 |
| **预约挂号** | 创建预约 | Redis+Lua原子扣号源，异步写库 | P0 | 核心亮点，改造原秒杀 |
| 预约挂号 | 防重复预约 | 同一医生同一天同一时段不能重复预约 | P0 | Lua Set校验 |
| 预约挂号 | 预约记录 | 查看我的预约（分页，按状态筛选） | P0 | 改造原订单 |
| 预约挂号 | 取消预约 | 号源回退+Lua原子操作 | P1 | 新增Lua脚本 |
| **评价** | 发表评价 | 填写评分+文字评价 | P1 | 改造原Blog |
| 评价 | 评价列表 | 按医院/医生查看评价（分页） | P1 | 改造原Blog |
| 评价 | 点赞评价 | Redis Set防重复点赞 | P2 | 改造原Blog |
| **关注** | 关注/取关 | 关注医院或医生 | P2 | 原项目未实现 |
| 关注 | 关注列表 | 查看关注列表（分页） | P2 | 原项目未实现 |

> **优先级定义：** P0 = 核心功能（MVP必须有），P1 = 重要功能，P2 = 锦上添花
>
> **复用说明：** "复用"表示在原项目对应功能基础上修改主题和字段
